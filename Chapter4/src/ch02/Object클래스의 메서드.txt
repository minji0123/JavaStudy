[equals() 메서드]
1. 같은 학번, 같은 사번, 같은 아이디의 회원
2. 두 인스턴스의 주소가 같은 곳을 가리키면 true, 아니면 false
3. 재정의를 해서, 두 인스턴스가 논리적으로 동일한지 판단할 수 있다.
    - 인스턴스가 다르더라도 논리적으로 동일한 경우, true 를 출력하게 재정의 가능

[hashCode() 메서드]
1. hashCode() 는 인스턴스의 저장 주소를 반환함
2. 힙메모리에 인스턴스가 저장되는 방식임
    - hash : 정보를 저장, 검색하는 자료구조
3. 자료의 특정 값(키 값)에 대한 저장 위치를 반환해주는 해시 함수를 사용
    - 자바에서 힙 메모리를 저장하는 방식이 해시방식임
    - 키 값이 있고, 이 키값에 대해서 오브젝트의 어디에다가 저장할 것인가
        - 오브젝트를 저장하는 방식이라고 생각하면 됨

equals()랑 hashCode() 두개가 같아야 같은 변수임

학생1 학생2가 있을 때 (둘은 다른 학생이지?)
그럼 학번을 hashCode()로 출력하면 됨
이름이 같아도 학번이 다르면 다른 학생이니까
데이터 구조의 기본키같은 애임

[clone() 메서드]
1. 객체의 원본을 복제하는데 사용하는 메서드
2. 근데 이거로 복제하면 객체 정보가 동일한 인스턴스들이 생기는 거여서
    객체 지향 프로그램에서의 정보은닉, 객체 보호의 관점에서 위배될 수 있음
3. cloneable : 해당 클래스의 clone() 메서드 사용 허용을 뜻함
