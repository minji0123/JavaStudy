[스트림 메소드]
1. 자료의 대상과 관계없이 동일한 연산을 수행
    - 배열, 컬렉션을 대상으로 연산 수행
    - '자료 처리에 대한 추상화가 구현되었다' 라고 함

2. 한번 생성하고 사용한 스트림은 재사용 불가
    - 자료에 대한 스트림을 생성하여 연산을 수행하면 스트림은 소모됨
        - 내부적으로 for 문이 돌고, 다 돌면 끝나기 때문에 다시 실행시켜야함
    - 자료에 대한 스트림을 수행하기 위해서는 스트림을 다시 생성해야 함

3. 스트림 연산은 기존 자료를 변경하지 않음
    - 자료에 대한 스트림 생성 -> 스트림이 사용하는 메모리 공간이 별도로 생성됨
    - 연산이 수행되어도 기존 자료가 변경되지 않음

4. 스트림 연산은 중간연산과 최종연산
    - 중간연산 : 중간중간 연산, 여러 개 연산이 적용 가능
        - filter(), map(), sorted()...
    - 최종연산 : 마지막에 한번만 적용
        - forEach(), count(), sum()...
    - 지연연산
        - 최종연산이 호출되어야 중간연산에 대한 수행이 이루어지고, 결과가 만들어짐
        - 따라서 중간연산에 대한 결과를 연산 중에 알 수 없음
        - sort
